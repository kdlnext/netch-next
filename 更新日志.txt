This release focuses on key protocol compatibility fixes, UI binding corrections, and the complete redesign of the Sing-box core automatic update workflow.

Updates:
- Updated application version to 2.1.2
- Fixed Trojan proxy failure by implementing a fallback mechanism that preserves the original hostname as SNI when a custom host is not configured
- Fixed TLS Allowinsecure option under the Singbox settings page to support proper check state binding, certificate verification skip, and persistence
- Restored and implemented full automatic Sing-box core download, extraction, safety process killing, and file replacement mechanism
- Added manual checking interactive flow with the "Manual Check Core Update" button in settings, showing download progress/success prompts
- Hardened manual check updates with a 10-second connection timeout and a fallback prompt offering manual download (official link) and step-by-step installation guidance (downloading 'windows-amd64' zip and overwriting 'sing-box.exe' in 'bin') if the check fails, times out, or detects no updates
- Added automated check and silent replacement upon startup when "Auto Update Sing-box" is enabled
- Configured "Auto Update Sing-box" to default to disabled and automatically reset to unchecked upon successful core replacement to avoid redundant downloads
- Added complete Chinese translations for all update-related steps in the UI

Improvements:
- Enhanced protocol handshake success rate under DNS-resolved IP scenarios for Trojan, VLESS, and VMess
- Provided seamless and secure Windows-side process locking release during core updates, preventing file access locked errors
- Refined settings data persistence behavior by syncing bound UI elements with immediately saved configuration files

Special thanks to all contributors.

本次版本重点修复了关键协议兼容性问题、修正了界面选项状态绑定，并全面实现了 Sing-box 内核的自动更新工作流。

更新内容：
- 程序版本更新为 2.1.2
- 修复了 Trojan 协议因启动前解析域名为 IP 导致 SNI 缺失、游戏提示网络异常的故障；引入 SNI 降级机制，在未配置 custom Host 时强制恢复为节点原始域名进行 TLS 握手
- 修复了设置页中“TLS Allowinsecure (允许不安全证书)”复选框无法勾选/保存的问题，实现数据绑定、证书验证跳过与状态持久化
- 恢复并实现了完整的 Sing-box 官方内核全自动下载、解压、安全进程锁释放与内核文件覆盖机制
- 在设置页中新增“手动检测内核更新”交互流程，支持发现新版本、弹窗确认、展示更新状态及成功提示
- 增强了手动检测的容错体验，设定 10 秒连接超时，并在网络超时/检测失败/已是最新版本时，提供手动下载官方最新内核的弹窗引导与步骤提示（下载 windows-amd64 包并解压覆盖 bin 目录同名文件）
- 实现了启动时静默检查并更新内核的后台逻辑（当勾选“自动更新 Sing-box”时触发）
- 将“自动更新 Sing-box”默认调整为不勾选，且在每次成功完成内核自动更新后，重置并自动取消勾选以避免重复拉取
- 补充了所有更新步骤对应的中文本地化翻译词条，提升用户交互体验

改进说明：
- 提升了 Trojan、VLESS、VMess 协议在域名被提前解析为 IP 时，TLS 握手阶段的连接成功率
- 优化了 Windows 下覆盖替换 `sing-box.exe` 时的文件锁定释放策略，自动关闭运行中的冲突内核进程，避免更新冲突
- 强化了设置项的持久化机制，界面状态变更后即时同步至本地配置文件，确保数据一致

感谢原始 Netch 项目作者及所有贡献者。
